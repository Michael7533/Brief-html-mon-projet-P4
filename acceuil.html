Méthodologie

les outils collaboratifs utilisés:
  • Teams pour la visio
  • IntelliJ pour saisir et tester le code2

la méthode mise en oeuvre:
  • Echanges quant à la cible du livrable
  • Brainstorming et formalisation préalable des fonctionnalités à intégrer (règles du jeu / alternance des joueurs / / méthode de resolution des test / ...)
  • Sessions de coding avec alternance des contributeurs dans le rôle de saisie du code
  • Lancement régulier du code pour valider le fonctionnement conforme aux attentes
  • Recherche

Modalités du projet

Vous allez réaliser ce projet en pair programming

Critères de performance

Le code utilise un tableau à 2 dimensions pour représenter le plateau de jeu. Les erreurs de saisie de l'utilisateur.ice sont gérées (colonne non existante, colonne pleine, ...) Le programme informe les joueurs lorsque le jeu se termine : qui gagne / qui perd

Résultat

le code est fonctionnel:

il permet de faire jouer alternativement les utilisateurs en leur demandant de selectionner la colonne dans laquelle ils souhaitent placer leur pion.

le pion est placé sur la première case non occupée de la colonne.

des tests sont réalisés pour rechercher un alignement des jetons.
si 4 pions identiques sont alignés, le gagnat est désigné. Dans le cas contraire la boucle recommence avec l'autre joueur.
